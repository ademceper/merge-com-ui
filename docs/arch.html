Monorepo & Package Management
Workspace orchestration, build cache, dependency yönetimi
pnpm v9+
2026'nın standart package manager'ı. Content-addressable store ile disk'e her paket yalnızca bir kez kaydedilir. Strict symlink tabanlı dependency tree — phantom dependencies imkansız. npm'den 3x, yarn'dan 2x hızlı kurulum.
ZORUNLU
Turborepo 2.x
Incremental build cache + remote caching. Dependency graph anlar — önce packages/ui build'lenir, sonra app'ler. Değişmeyen paketler cache'den <1 saniyede replay edilir. Paralel task execution.
ZORUNLU
Nx alt.
Turborepo'ya güçlü alternatif. Code generation, affected builds, project graph UI. 10+ developer'lı kurumsal ekipler için tercih edilebilir. Turborepo'dan daha fazla konfigürasyon gerektirir.
ALTERNATİF
Syncpack
Monorepo'daki tüm workspace'lerde aynı paket versiyonu zorunluluğu. "Version drift" sorununu otomatik tespit eder — iki farklı yerde React yüklenirse bundle şişer, context hataları çıkar.
ÖNERİLEN
Changesets
Internal package versioning ve otomatik CHANGELOG üretimi. packages/ui'yi değiştirdiğinde hangi app'lerin etkileneceğini takip eder. Semantic versioning disiplini.
ÖNERİLEN
turbo.json // Turborepo pipeline konfigürasyonu { "$schema": "https://turborepo.com/schema.json", "tasks": { "build": { // Önce bağımlı paketleri build et "dependsOn": ["^build"], "outputs": [".next/**", "dist/**"], "inputs": ["$TURBO_DEFAULT$", ".env*"] }, "dev": { "cache": false, "persistent": true }, "lint": { "dependsOn": ["^lint"] }, "test": { "dependsOn": ["^build"] }, "type-check": { "dependsOn": ["^build"] } } }

Developer Tooling

🦀

Biome v2

ESLint + Prettier'ın 2026'daki halefi. Rust ile yazıldı — 10-25x daha hızlı. Tek binary, tek config. Haziran 2025'te v2.0 yayınlandı: plugin API ve type-aware linting eklendi. TypeScript-eslint'in yüzde 85 kapsamı.

// biome.json — tek dosya yeterli { "linter": { "rules": { "recommended": true } }, "formatter": { "indentStyle": "space", "indentWidth": 2 }, "javascript": { "formatter": { "quoteStyle": "single" } } }
📘

TypeScript 5.x

strict: true zorunlu. Tüm packages ve apps ortak bir tsconfig.base.json'dan extend eder. paths ile @ui/*, @store/*, @hooks/* alias'ları tüm workspace'te tutarlı çalışır.

// packages/config/tsconfig.base.json { "compilerOptions": { "strict": true, "target": "ES2022", "moduleResolution": "bundler", "verbatimModuleSyntax": true } }
🪝

Git Hooks & Commit

  • Lefthook — Git hooks manager. Husky'den çok daha hızlı (binary, Go ile yazıldı). Parallel hook execution.
  • Commitlint — Conventional Commits zorunluluğu. feat:, fix:, chore: pattern'ları.
  • lint-staged — Yalnızca staged dosyaları lint'le, tüm projeyi değil. Pre-commit hızlanır.
🔄

Alternatif: oxlint + Prettier

Pure linting hızı istiyorsan: oxlint ESLint'ten 50x daha hızlı. Ama formatting için Prettier'a ihtiyaç var. Biome'dan daha az kapsam, fakat edge case'lerde daha esnek. Büyük legacy ESLint config varsa bu yol tercih edilebilir.

⚠️
Biome Sınırı: HTML, Markdown, SCSS, Vue ve Astro dosyalarına destek hâlâ kısmi. Bu dosya tiplerine yoğun çalışıyorsan Biome + ESLint karma yaklaşımı daha güvenli.

Ne zaman ne kullan?

04
Admin/Dashboard — React 19 + Vite 6
React Compiler, Actions API, TanStack Router
⚛️

React 19 — React Compiler

React Compiler (eski adıyla React Forget) ile otomatik memoization. useMemo, useCallback, memo büyük ölçüde gereksiz hale geldi. Erken benimseyenler yüzde 25-40 daha az re-render bildiriyor. Instagram production'da kullanıyor.

🎬

React 19 — Actions API

Async fonksiyonlar artık transition içinde çalışır. useActionState, useOptimistic, useFormStatus — form ve async işlemler drastik biçimde sadeleşti. Basit formlar için artık React Hook Form şart değil.

const [state, action, isPending] = useActionState( submitForm, null )
🔗

TanStack Router v2

React Router v7'den üstün type-safety. Route parametreleri, search params, context — hepsi fully-typed. File-based routing + code-based karma. Vite plugin ile route config otomatik üretilir. Built-in SWR caching.

⚡

Vite 6

Native ESM, anında cold start, sub-millisecond HMR. Environment API (v6 yeni) ile server/client ortamları net ayrılır. Rollup tabanlı production build, mükemmel tree-shaking ve code splitting.

TanStack Router v2 vs React Router v7 — Detaylı Karşılaştırma

Özellik TanStack Router v2 React Router v7 Karar
TypeScript 100% type-safe, ground-up TS Framework mode'da iyi, library mode'da sınırlı TanStack
Search Params JSON, schema validation, immutable String tabanlı, manuel parsing TanStack
Data Loading Built-in loaders + SWR cache Loader functions (v7) Berabere
File-based Routing Vite plugin ile otomatik Framework mode gerektirir TanStack
Ekosistem Genç ama hızlı büyüyor On yıllık, geniş community React Router
Mevcut Proje Migration gerektirir v6→v7 non-breaking (flag'larla) React Router
✅
2026 Önerisi: Yeni Vite projesi başlatıyorsan TanStack Router v2. Mevcut React Router projesi varsa v7'ye upgrade et. Her iki durumda da Next.js Pages Router'ı yeni projede kullanma.

Persistence — Platform Bazlı

Web

  • zustand/middleware persist → localStorage / sessionStorage
  • TanStack Query persist → IndexedDB ile büyük cache kalıcılığı
  • Cookies (httpOnly) → Auth token, güvenli

Mobile

  • MMKV → AsyncStorage'dan 10x hızlı, synchronous, encrypted
  • Expo SecureStore → Token, sensitive data, Keychain/Keystore
  • Expo FileSystem → Binary, büyük dosyalar

Data Fetching & API Katmanı
HTTP client, GraphQL, Realtime, Mock, API client paketi
🌐

ky

Fetch API wrapper. Axios'tan 3x küçük (~4KB), modern API. Retry, timeout, hooks, JSON otomatik parse. Node.js + browser + React Native universal. beforeRequest/afterResponse hooks ile merkezi auth header yönetimi.

🌍

ofetch

Nuxt/Nitro ekibinden alternatif HTTP client. Daha zengin API, auto-retry, timeout, interceptors. Edge runtime uyumlu. ky yerine tercih edilebilir, API çok benzer.

📊

GraphQL — urql

Apollo'dan daha hafif ve esnek. Exchange tabanlı mimari — cache, auth, retry her biri ayrı exchange. Normalized cache, Suspense desteği. Büyük API'ler için güçlü seçim.

⚡

Realtime

  • WebSocket native → basit, hiç bağımlılık yok
  • socket.io-client → fallback, room, namespace
  • EventSource (SSE) → AI streaming API'ları için ideal
  • Ably / Pusher → hosted realtime servis
🎭

MSW v2 — Mock Service Worker

Service Worker tabanlı API mock. Network seviyesinde intercept — gerçek HTTP requestleri. Hem development hem test ortamında aynı mock. Browser + Node.js. en gerçekçi mock çözümü.

📦

packages/api-client

Monorepo'daki merkezi API katmanı. ky üzerine kurulu, type-safe endpoint'ler. Tüm app'ler bu paketi tüketir — auth header, base URL, error handling tek yerde.

2026'nın 5 Oyun Değiştirici Kararı

🦀

1 — Tailwind v4 Oxide Engine

Rust tabanlı yeniden yazım. Incremental build 100x hızlı (44ms → 5ms). CSS-first config — tailwind.config.js artık yok. OKLCH renk uzayı, container queries built-in. Yeni projeler için geçiş zorunlu.

⚛️

2 — React 19 Compiler

Otomatik memoization — useMemo, useCallback, memo büyük ölçüde gereksiz. Erken benimseyenler yüzde 25-40 daha az re-render bildiriyor. Actions API ile form ve async işlemler sadeleşti.

🚨

3 — Expo New Architecture Zorunlu

Legacy Architecture Haziran 2025'te donduruldu. SDK 55+'ta kapatılamaz. Büyük kütüphaneler (Reanimated v4, Flash List v2) New Architecture only. SDK 53+ ile başlamak zorunlu.

🦀

4 — Biome v2 — ESLint+Prettier Bitti

Tek binary, tek config dosyası — ESLint + Prettier + onlarca plugin yerine. 10-25x daha hızlı. Haziran 2025'te v2: plugin API + type-aware linting. Monorepo CI time'ı dramatik düşer.

🔄

5 — TanStack Query v5 Fiili Standart

v4'ten yüzde 20 küçük. Unified API, isPending, useSuspenseQuery, queryOptions. Server state yönetimi için artık alternatifsiz. Web ve mobile'da aynı hook — packages/hooks'ta paylaşılır.

🔁

6 — TanStack Router — Type-Safe Routing

React Router v7 yaklaşmış ama type-safety'de TanStack hâlâ önde. Route params, search params, context — hepsi fully-typed. Vite projelerinde artık varsayılan tercih olmalı.
